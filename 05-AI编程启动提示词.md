# 滴兔智能体 AI 编程启动提示词

以下提示词用于驱动 AI 编程助手直接落地“滴兔智能体”完整工程。AI 必须先读取本仓库 6 份文档，再执行代码生成、测试和修正。

## 1. 总启动提示词

```text
你是资深 Java 17 + Spring Boot 3.2 + Vue 3 全栈工程师。请在当前仓库中为“滴兔知识产权有限公司”实现“滴兔智能体”完整工程。

必须先读取以下文档：
- 00-README-文档索引.md
- 01-功能清单与业务范围.md
- 02-技术架构设计说明书.md
- 03-数据库与接口契约说明书.md
- 04-工程实施与验收标准.md
- 05-AI编程启动提示词.md

功能基线：
- 01-功能清单与业务范围.md 是唯一功能基线。
- 所有开发任务必须引用功能编号。
- 新增接口、表、页面、测试前必须确认对应功能编号。
- 功能实现完成后必须回填对应功能编号的测试结果。
- 代码实现必须覆盖 `DEV-001`，Java、TypeScript、SQL 都必须补充有效业务注释。

业务范围：
- 用户端支持账号密码登录、多轮会话、聊天记录、套餐查看、专属客服、工单。
- 管理端支持管理员登录、用户注册、用户停用、用户启用、套餐次数分配、次数控制、用户大模型链接配置、工单处理、RAG 知识维护。
- 系统支持用户小程序、用户 H5、管理端 Web。

后端技术：
- Java 17。
- Spring Boot 3.2+。
- Spring Security 6.x。
- Spring WebFlux SSE。
- MyBatis Plus 3.x。
- PostgreSQL 15+。
- pgvector。
- AgentScope Java 依赖 io.agentscope:agentscope:1.0.12。
- Maven 3.8+。
- Testcontainers 1.19+。

前端技术：
- Vue 3。
- TypeScript。
- Vite。
- Element Plus。
- Pinia。
- SSE Client。
- Markdown 渲染组件。

工程规则：
- 后端只创建一个 Maven 工程 backend。
- 后端只创建一个 Spring Boot 应用。
- 后端禁止拆分多个 Maven 子模块。
- Java 代码通过包分层实现 common、auth、user、chat、agent、rag、ticket、admin、infra。
- 平台默认大模型链接使用 DITU_DEFAULT_MODEL_BASE_URL、DITU_DEFAULT_MODEL_NAME、DITU_DEFAULT_MODEL_AUTH_TYPE、DITU_DEFAULT_MODEL_API_KEY。
- 管理端必须支持为每个用户配置一条大模型链接。
- Agent run 必须优先使用当前用户启用的大模型链接。
- 当前用户没有启用的大模型链接时，Agent run 使用平台默认大模型链接。
- 模型密钥必须加密存储，接口响应不得返回明文密钥。
- Embedding 模型单独配置为 DITU_EMBEDDING_MODEL。
- Controller 和业务服务不得直接引用 AgentScope 类。
- chat 包只能通过 AgentRuntimePort 调用 Agent。
- AgentScope 代码只能存在于 agent/adapter 包。
- 次数控制必须写 quota_ledger。
- 管理端敏感操作必须写 audit_log。
- RAG 检索必须过滤 GLOBAL 和当前用户 USER 知识。
- 代码必须补充注释，注释必须解释业务规则、权限边界、事务边界、RAG 权限过滤、次数扣减、模型链接解析、SSE 事件语义和异常处理原因。
- 禁止使用“获取数据”“设置值”这类没有业务含义的机械注释。

工作方式：
1. 检查当前仓库结构。
2. 保留用户已有代码。
3. 按实施步骤创建文件。
4. 每个实施步骤运行测试命令。
5. 测试失败时修复后重新运行。
6. 最终输出变更文件、测试命令、测试结果、未完成项。

第一个实施步骤只完成工程骨架：
- 覆盖 `OPS-001`、`OPS-002`、`OPS-003`、`DEV-001` 的工程基础。
- backend 单 Maven 工程。
- Spring Boot 启动应用。
- Java 包分层。
- Flyway。
- PostgreSQL/Testcontainers。
- frontend Vue 3 Monorepo。
- deploy/docker-compose.yml。
- README 启动说明。
```

## 10. 禁止事项

- 禁止创建多个后端 Maven 子模块。
- 禁止让 Controller 直接引用 AgentScope 类。
- 禁止让业务服务直接引用 AgentScope 类。
- 禁止绕过 `quota_ledger` 修改次数。
- 禁止 RAG 检索遗漏用户权限过滤。
- 禁止用户端接口返回其他用户数据。
- 禁止用户端接口返回服务器文件路径。
- 禁止模型密钥明文落库。
- 禁止模型密钥出现在日志、SSE、普通接口响应中。
- 禁止提交真实密钥、数据库密码、模型 API Key。
