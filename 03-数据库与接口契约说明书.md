# 滴兔智能体数据库与接口契约说明书

## 1. 契约规则

- 本文档以 [01-功能清单与业务范围.md](01-功能清单与业务范围.md) 为功能基线。
- 每个接口必须服务至少一个功能编号。
- 新增接口前先在功能清单中确认功能编号。
- API 前缀统一为 `/api/v1`。
- 管理端 API 前缀统一为 `/api/v1/admin`。
- 普通 JSON 接口返回统一响应包。
- SSE 接口返回 `text/event-stream`。
- 时间字段使用 `timestamptz` 落库，接口使用 ISO-8601 字符串。
- 主键使用 `bigserial`。
- 数据库使用 PostgreSQL 15+。
- RAG 向量检索使用 pgvector，向量字段固定为 `vector(1536)`。
- 用户次数变化必须写入 `quota_ledger`。
- 管理端敏感操作必须写入 `audit_log`。
- 用户只能访问自己的会话、消息、工单和专属知识。
- Agent run 启动前必须解析当前用户启用的大模型链接。
- 用户未配置或模型链接停用时，Agent run 使用平台默认模型链接。
- 数据库迁移脚本必须为核心表和核心字段补充 `COMMENT ON TABLE`、`COMMENT ON COLUMN` 注释。
- Mapper SQL 必须补充说明业务规则、权限条件、事务意图和异常边界的注释。

### 3.1 用户表

```sql
CREATE TABLE app_user (
  id                BIGSERIAL PRIMARY KEY,
  username          VARCHAR(64) UNIQUE NOT NULL,
  phone             VARCHAR(32),
  email             VARCHAR(128),
  password_hash     VARCHAR(255) NOT NULL,
  display_name      VARCHAR(80) NOT NULL,
  avatar_url        VARCHAR(512),
  role_code         VARCHAR(32) NOT NULL DEFAULT 'USER',
  status            VARCHAR(32) NOT NULL DEFAULT 'ACTIVE',
  plan_code         VARCHAR(32) NOT NULL DEFAULT 'BASIC',
  quota_total       INTEGER NOT NULL DEFAULT 0,
  quota_used        INTEGER NOT NULL DEFAULT 0,
  quota_reset_at    TIMESTAMPTZ,
  wechat_openid     VARCHAR(128),
  last_login_at     TIMESTAMPTZ,
  created_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
  CONSTRAINT ck_app_user_role CHECK (role_code IN ('USER', 'ADMIN', 'RAG_ADMIN', 'CS_MANAGER')),
  CONSTRAINT ck_app_user_status CHECK (status IN ('ACTIVE', 'DISABLED', 'LOCKED')),
  CONSTRAINT ck_app_user_quota CHECK (quota_total >= 0 AND quota_used >= 0 AND quota_used <= quota_total)

CREATE INDEX idx_app_user_phone ON app_user(phone);
CREATE INDEX idx_app_user_status ON app_user(status);

```

字段规则：

- `base_url` 存储大模型服务基础地址。
- `model_name` 存储调用模型名称。
- `api_key_ciphertext` 存储加密后的密钥。
- 管理端查询接口不返回 `api_key_ciphertext`。
- 一个用户最多拥有一条模型链接配置。

## 5. 错误码

| 错误码 | HTTP | 含义 |
|---|---:|---|
| `UNAUTHORIZED` | 401 | 未登录或 Token 失效 |
| `FORBIDDEN` | 403 | 无权限 |
| `VALIDATION_ERROR` | 400 | 参数错误 |
| `USER_DISABLED` | 403 | 用户已停用 |
| `QUOTA_NOT_ENOUGH` | 409 | 次数不足 |
| `CONVERSATION_NOT_FOUND` | 404 | 会话不存在或无权访问 |
| `AGENT_RUN_FAILED` | 500 | Agent 执行失败 |
| `MODEL_CONFIG_NOT_FOUND` | 404 | 用户模型链接配置不存在 |
| `MODEL_CONNECTION_FAILED` | 422 | 用户模型链接连通性测试失败 |
| `RAG_DOCUMENT_FAILED` | 422 | 文档解析或入库失败 |
| `TICKET_NOT_FOUND` | 404 | 工单不存在或无权访问 |

### 6.8 订阅 SSE

`GET /api/v1/conversations/{conversationId}/runs/{runId}/events`

响应类型：`text/event-stream`

事件示例：

```text
event: run.started
id: 1
data: {"runId":9001,"status":"RUNNING"}

event: message.delta
id: 2
data: {"runId":9001,"delta":"注册商标通常需要先进行近似检索，"}

event: rag.context
id: 3
data: {"runId":9001,"chunks":[{"documentName":"商标注册FAQ.pdf","score":0.82,"snippet":"申请前先进行近似检索。"}]}

event: message.done
id: 4
data: {"runId":9001,"messageId":102,"content":"注册商标通常需要先进行近似检索，然后准备主体资格材料、商标图样和商品服务类别。"}

event: quota.updated
id: 5
data: {"runId":9001,"quotaCost":1,"remainingQuota":287}

event: run.completed
id: 6
data: {"runId":9001,"status":"COMPLETED"}
```

SSE 客户端规则：

- 收到 `message.delta` 后追加当前助手消息。
- 收到 `message.done` 后使用完整内容覆盖最终助手消息。
- 收到 `error` 后展示错误并结束当前 run。
- 断线重连时发送 `Last-Event-ID`，后端按 `agent_run_event.sequence_no` 补发事件。

#### 保存用户模型链接

`PUT /api/v1/admin/users/{userId}/model-config`

覆盖功能：`ADMIN-MODEL-002`

请求：

```json
{
  "configName": "客户自有模型",
  "providerCode": "CUSTOM",
  "baseUrl": "https://llm.example.com/v1",
  "modelName": "customer-model",
  "authType": "API_KEY",
  "apiKey": "sk-***",
  "enabled": true
}
```

处理规则：

- `baseUrl` 必须是 `http` 或 `https` 地址。
- `modelName` 不能为空。
- `authType` 为 `API_KEY` 或 `BEARER` 时，首次保存必须提交 `apiKey`。
- `apiKey` 加密后写入 `api_key_ciphertext`。
- 请求未提交 `apiKey` 时保留原密钥密文。
- 保存成功后写入 `audit_log`。
- 保存后该用户下一次 Agent run 使用该配置。

## 8. 内部 RAG 检索契约

方法：

```text
RagSearchResult search(RagSearchCommand command)
```

命令字段：

| 字段 | 含义 |
|---|---|
| `userId` | 当前用户 ID |
| `query` | 用户问题 |
| `topK` | 召回数量，固定默认 5 |
| `minScore` | 最低相似度 |
| `includeGlobal` | 是否包含通用知识 |
| `includeUserOwned` | 是否包含用户专属知识 |

SQL 权限条件：

```sql
SELECT c.*
FROM rag_chunk c
JOIN rag_collection rc ON rc.id = c.collection_id
JOIN rag_document d ON d.id = c.document_id
WHERE rc.enabled = true
  AND d.status = 'READY'
  AND (
    rc.scope = 'GLOBAL'
    OR (rc.scope = 'USER' AND rc.owner_user_id = :currentUserId)
  )
ORDER BY c.embedding <=> :queryEmbedding
LIMIT :topK;
```

## 9. 内部模型链接解析契约

方法：

```text
ModelConnection resolveForUser(Long userId)
```

解析规则：

1. 查询 `app_user_model_config` 中 `user_id = :userId AND enabled = true` 的配置。
2. 查询到用户配置时返回用户级 `baseUrl`、`modelName`、`authType`、解密后的密钥。
3. 查询不到用户配置时返回平台默认模型链接。
4. 平台默认模型链接来自环境变量 `DITU_DEFAULT_MODEL_BASE_URL`、`DITU_DEFAULT_MODEL_NAME`、`DITU_DEFAULT_MODEL_AUTH_TYPE`、`DITU_DEFAULT_MODEL_API_KEY`。
5. 返回对象只在内存中携带解密密钥，不写日志，不写 SSE，不写普通业务响应。

Agent run 记录规则：

- `agent_run.model_config_id` 记录用户级配置 ID。
- 使用平台默认模型链接时 `agent_run.model_config_id` 为空。
- `agent_run.model_name` 记录本次实际使用的模型名称。
- 模型调用失败时写入 `agent_run.error_code = 'MODEL_CONNECTION_FAILED'`。

## 10. 核心业务规则

### 10.1 次数

- 用户状态不是 `ACTIVE` 时禁止发起会话。
- `quota_total - quota_used <= 0` 时返回 `QUOTA_NOT_ENOUGH`。
- 一次用户消息消耗 1 次。
- Agent 成功完成后提交消耗。
- Agent 失败、超时、取消后回滚预占。
- 管理员调整次数时写入 `quota_ledger` 和 `audit_log`。

### 10.2 会话

- 用户只能访问自己的 `conversation`、`chat_message`、`agent_run`。
- 管理端查看会话内容必须通过独立权限控制。
- 会话删除采用 `status = 'DELETED'` 软删除。
- 会话运行时必须先解析用户模型链接，再启动 Agent。
- 用户模型链接调用失败时本次 Agent run 失败，次数回滚。

### 10.3 用户模型链接

- 用户模型链接只能由 `ADMIN` 维护。
- 模型链接密钥必须加密存储。
- 管理端查询接口必须脱敏，不返回明文密钥。
- 保存、测试、停用模型链接必须写入 `audit_log`。
- 一个用户最多存在一条模型链接配置。
- 启用配置优先于平台默认模型链接。
- 停用配置后使用平台默认模型链接。

### 10.4 工单

状态流转：

```text
OPEN -> PENDING -> PROCESSING -> RESOLVED -> CLOSED
OPEN -> PROCESSING
RESOLVED -> PROCESSING
```

规则：

- `CLOSED` 工单禁止追加消息。
- 用户追加消息时，`RESOLVED` 工单回到 `PROCESSING`。
- 工单分配、状态变化、关闭写入 `audit_log`。

### 10.5 RAG 文档

状态流转：

```text
UPLOADED -> PARSING -> READY
UPLOADED -> PARSING -> FAILED
FAILED -> PARSING
READY -> DISABLED
```

规则：

- 只有 `READY` 文档参与检索。
- `DISABLED` 文档不参与检索。
- 用户专属知识只被所属用户召回。
