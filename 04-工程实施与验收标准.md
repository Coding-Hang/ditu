# 滴兔智能体工程实施与验收标准

## 1. 工程目标

本工程交付一个完整可运行系统：

- 用户小程序：覆盖 `AUTH`、`USER`、`CHAT`、`CS`、`TICKET` 功能。
- 用户 H5：覆盖 `AUTH`、`USER`、`CHAT`、`CS`、`TICKET` 功能。
- 管理端 Web：覆盖 `ADMIN-USER`、`ADMIN-QUOTA`、`ADMIN-PLAN`、`ADMIN-MODEL`、`ADMIN-TICKET`、`RAG` 功能。
- 后端：一个 Maven 工程、一个 Spring Boot 应用、Java 包分层、PostgreSQL、SSE、AgentScope Java、Testcontainers。

本文档以 [01-功能清单与业务范围.md](01-功能清单与业务范围.md) 为验收基线。实施步骤、测试用例和最终验收清单全部按功能编号追踪。

模型规则：

- 平台配置默认大模型链接，环境变量为 `DITU_DEFAULT_MODEL_BASE_URL`、`DITU_DEFAULT_MODEL_NAME`、`DITU_DEFAULT_MODEL_AUTH_TYPE`、`DITU_DEFAULT_MODEL_API_KEY`。
- 管理端为每个用户配置一条启用的大模型链接。
- Agent run 优先使用当前用户启用的大模型链接。
- 当前用户没有启用的大模型链接时，Agent run 使用平台默认大模型链接。
- Embedding 模型单独配置为 `DITU_EMBEDDING_MODEL`。
- 模型密钥加密存储，接口响应不返回明文密钥。

## 4. 实施步骤

### 步骤 0：工程骨架

交付内容：

- `backend` 单 Maven 工程。
- Spring Boot 启动类 `DituAgentApplication`。
- Java 包：`common`、`auth`、`user`、`chat`、`agent`、`rag`、`ticket`、`admin`、`infra`。
- Flyway 数据库迁移目录。
- PostgreSQL 15+ Docker Compose。
- Testcontainers PostgreSQL 集成测试基类。
- `frontend` Vue 3 Monorepo。
- H5、管理端、小程序三个应用目录。

验收标准：

- `mvn clean test` 通过。
- Spring Boot 应用启动成功。
- `/actuator/health` 返回 `UP`。
- Testcontainers 成功启动 PostgreSQL。
- `pnpm install` 成功。
- `pnpm typecheck` 通过。

### 步骤 1：登录、用户、套餐、次数、用户模型链接

交付内容：

- 覆盖 `AUTH-001` 至 `AUTH-003`、`USER-001`、`USER-002`、`ADMIN-USER-001` 至 `ADMIN-USER-005`、`ADMIN-QUOTA-001` 至 `ADMIN-QUOTA-003`、`ADMIN-PLAN-001`、`ADMIN-PLAN-002`、`ADMIN-MODEL-001` 至 `ADMIN-MODEL-004`、`DEV-001`。
- `app_user`、`subscription_plan`、`app_user_model_config`、`quota_ledger`、`audit_log` 迁移脚本。
- 基础、Pro、Plus 三个套餐初始化数据。
- 登录、当前用户、刷新 Token。
- 管理端用户注册、停用、启用。
- 管理端套餐和次数分配。
- 管理端用户大模型链接查询、保存、测试、停用。
- 次数流水查询。
- 用户端套餐展示。
- 管理端用户管理页面。

验收标准：

- 用户登录成功后返回 Token、套餐、总次数、已用次数、剩余次数。
- 停用用户无法登录。
- 停用用户无法发起会话。
- 管理员调整次数后写入 `quota_ledger`。
- 管理员调整次数后写入 `audit_log`。
- 管理员保存用户大模型链接后写入 `audit_log`。
- 管理端查询用户大模型链接时不返回明文密钥。
- 用户模型链接测试成功后记录 `SUCCESS`。
- 停用用户模型链接后该用户回退平台默认模型链接。
- 用户端套餐页显示基础、Pro、Plus。

### 步骤 2：会话、消息、SSE

交付内容：

- 覆盖 `CHAT-001` 至 `CHAT-008`、`AGENT-002`、`AGENT-003`、`AGENT-007`、`DEV-001`。
- `conversation`、`chat_message`、`agent_run`、`agent_run_event` 迁移脚本。
- 会话创建、会话列表、消息分页。
- 发送消息创建 `agent_run`。
- `agent_run` 记录本次使用的 `model_config_id` 和 `model_name`。
- WebFlux SSE 输出事件。
- Mock Agent 输出固定流式回复。
- SSE 事件落库。
- `Last-Event-ID` 补发。
- 成功扣减次数，失败回滚次数。

验收标准：

- 用户完成 3 轮连续会话。
- 用户刷新页面后查看历史消息。
- 用户从历史会话继续发送消息。
- 次数不足时发送消息返回 `QUOTA_NOT_ENOUGH`。
- Agent 成功后剩余次数减少 1。
- Agent 失败后剩余次数不变。
- SSE 事件顺序为 `run.started`、`message.delta`、`message.done`、`quota.updated`、`run.completed`。

### 步骤 3：AgentScope Java 适配

交付内容：

- 覆盖 `AGENT-001` 至 `AGENT-007`、`ADMIN-MODEL-001` 至 `ADMIN-MODEL-004`、`DEV-001`。
- `agent/port/AgentRuntimePort`。
- `agent/core/AgentRunCommand`。
- `agent/core/AgentEvent`。
- `agent/tool/ToolRegistry`。
- `agent/skill/SkillDefinition`。
- `agent/memory/ConversationMemoryStore`。
- `agent/llm/ModelConnectionResolver`。
- `agent/llm/ModelConnection`。
- `agent/hitl/HitlGateway`。
- `agent/adapter/AgentScopeRuntimeAdapter`。
- `agent/adapter/MockAgentRuntimeAdapter`。

验收标准：

- `chat` 包只依赖 `AgentRuntimePort`。
- Controller 不引用 AgentScope 类。
- Application Service 不引用 AgentScope 类。
- `AgentScopeRuntimeAdapter` 将 AgentScope 输出转换为 `AgentEvent`。
- Mock Agent 和 AgentScope Adapter 使用同一接口。
- 用户 A 和用户 B 配置不同模型链接时，两人的 Agent run 使用各自配置。
- 用户模型链接停用后，Agent run 使用平台默认模型链接。
- 用户模型链接调用失败时 Agent run 失败并回滚次数。

验收标准：

- 用户从客服页创建工单成功。
- 用户从会话页转人工创建工单成功。
- 管理端按状态筛选工单。
- 管理端分配工单成功。
- 管理端回复工单成功。

## 8. 安全标准

- 密码不明文存储。
- Token 密钥不提交到仓库。
- 管理端接口执行角色校验。
- 用户端接口执行数据归属校验。
- 文件上传限制大小、扩展名、MIME。
- 普通用户接口不返回服务器文件路径。
- 模型密钥加密存储。
- 模型密钥不进入日志、SSE、普通接口响应。
- RAG 检索执行 `GLOBAL OR owner_user_id = current_user.id`。
- 敏感管理操作写入 `audit_log`。
