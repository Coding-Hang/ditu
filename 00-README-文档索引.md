## 执行顺序

1. 读取本索引，确认文档边界和技术基线。
2. 读取 [01-功能清单与业务范围.md](01-功能清单与业务范围.md)，确认功能编号和交付边界。
3. 读取 [02-技术架构设计说明书.md](02-技术架构设计说明书.md)，建立统一架构理解。
4. 读取 [03-数据库与接口契约说明书.md](03-数据库与接口契约说明书.md)，按接口和数据契约并行开发。
5. 读取 [04-工程实施与验收标准.md](04-工程实施与验收标准.md)，按实施步骤交付、测试、验收。
6. 使用 [05-AI编程启动提示词.md](05-AI编程启动提示词.md) 驱动 AI 编程助手生成工程代码。

## 交付范围

用户端范围：

- 账号密码登录。
- 多轮会话。
- 聊天记录查看和继续会话。
- 基础、Pro、Plus 套餐查看。
- 专属客服列表查看。
- 用户发起和查看专属客服工单。

管理端范围：

- 管理员账号密码登录。
- 用户注册、停用、启用。
- 用户套餐和付费次数分配。
- 用户会话次数控制。
- 用户大模型链接配置。
- 专属客服工单列表、分配、回复、关闭。
- 通用 RAG 知识维护。
- 单一用户专属 RAG 知识维护。

三端范围：

- 用户小程序。
- 用户 H5。
- 管理端 Web。

后端范围：

- 单 Maven 工程。
- 一个 Spring Boot 应用。
- Java 包分层承载 auth、user、chat、agent、rag、ticket、admin、infra 能力。
- 不拆分多个 Java 子模块。

## 交付规则

- 设计文档使用确定性表达，避免让执行方自行判断架构方向。
- 功能变更先修改 [01-功能清单与业务范围.md](01-功能清单与业务范围.md)，再同步修改其他文档。
- Java 后端只保留一个 Maven 工程和一个 Spring Boot 启动应用。
- 业务层通过 `AgentRuntimePort` 调用 Agent 能力，Controller 和业务服务不直接引用 AgentScope 内部类。
- Agent run 按用户解析大模型链接，用户未启用专属配置时使用平台默认模型链接。
- Java、TypeScript、SQL 代码必须补充有效业务注释。
- 所有次数变化写入流水。
- 所有 RAG 检索执行权限过滤。
- 所有管理端敏感操作写入审计日志。
